Hi, I'm Elena Vera Moser!

Software Engineer and Linguist

Originally from Switzerland, I have grown up in the canton of Bern in a bilingual French-German setting. Early on I have been confronted with different linguistic systems and developed a passion for language learning. In 2017, I completed a Bachelor in General Linguistics at the University of Bern, and in 2019, I graduated from Stockholm University with a Master in Typology and Linguistic Diversity. Besides being fluent in in German, French, and English, I have also acquired proficiency in Swedish since moving to Stockholm in 2017. Moreover, I speak Spanish and some Mandarin Chinese. Currently, I am learning Farsi.

Alongside my passion for human languages, I have developed a keen interest in computer languages and enjoy creating beautiful and functional applications. My journey into coding began while working as a linguist on a Data Science team at Gavagai (which has since ceased operations). Writing SQL and Python programs to clean, parse, and update big data sets was an integral part of my job. Over time, I took on more programming projects, using SQL, Python, Git, and Docker daily to assist my colleagues with expanding and improving Gavagai's flagship product – a review analysis system capable of processing customer feedback across more than 40 languages. In 2022, I pursued formal education in Computer Science with a focus on Web Development at Blekinge Tekniska Högskolan, aiming to become a certified full-stack developer and launch a career in tech.

Portfolio

As a full-stack developer specializing in web technologies, I have contributed to both the backend and frontend of numerous web applications. Below is a selection of projects I have developed, either independently or in collaboration with a teammate.

Single Page Application for Mobile Devices

This individual examination project for the course Web Applications for Mobile Devices focused on responsive design. The application provides users with real-time information about train delays within Sweden, utilizing data accessed through the Swedish Transportation Administration's (Trafikverket) API. Developed as a single-page application, the project was built using JavaScript and NodeJS with Express.

Web Application using Vue and Sockets

This team project was part of the JavaScript-based Web Framworks course, which focused on gaining proficiency with web frameworks. The application provides users with up-to-date information on train traffic across Sweden, pulling data from the Swedish Transportation Administration (Trafikverket) API. This web application was developed with JavaScript, Vue and NodeJS Express. For real-time updates of train delays we used socket.io, a tool which provides a bi-directional communication channel between a client and a server.

Data Analysis with Python (private repo)

This individual examination project was part of the Mathematical Modelling course. I used Python, along with libraries such as pandas and scipy, to analyze sunshine duration data for Stockholm, Karlskrona, and Gothenburg. Additionally, I utilized streamlit.io to create interactive visualizations, allowing for clear and intuitive presentation of the data and results.

Client Website

This individual project was part of the Technical Web Design and Usability course, where I developed a web application for a fictional client—in this case, my younger brother, an aspiring musician. The course emphasized accessible and user-friendly design, allowing me to apply key principles such as hierarchy, composition, and depth. I created two versions of the website, offering both light and dark modes to enhance usability and cater to different user preferences.

Thesis Project in Web Content Accessibility

For my final thesis project at the Blekinge Institute of Technology (BTH), my teammate and I collaborated with the Web/Mobile team at the IT Services Office (ID) at Bern University. We explored a selection of open-source web accessibility evaluation tools and investigated how easily they could be integrated into a CI/CD environment. The study focused on assessing the effectiveness of these tools in detecting WCAG violations within a CI/CD pipeline. The tools selected for our research included Asqatasun, Deque Systems’ Axe-Core CLI, Squiz Labs’ HTML_CodeSniffer, and IBM’s Equal Access Accessibility Checker. To conduct a thorough evaluation, we developed a test website specifically designed with a set number of Web Content Accessibility Guidelines (WCAG) violations. The findings of our study provided valuable insights that enabled the Web/Mobile team to rethink and enhance their CI/CD pipeline for better accessibility compliance.
